# NewChat 本地 Agent 说明

## 开发与测试

- UI 测试以宽屏桌面状态为准；宽屏状态没问题时，不需要额外测试竖屏/移动窄屏。
- 需要在浏览器里测试项目时，必须使用 `pnpm dev` 启动完整前后端，不要直接运行 `vite`，避免前端代理到其他 worktree 的后端进程。
- 测试大模型调用时不需要为了节省额度切换到 Mock 模式，按当前真实模型配置验证即可。

## 人物展示素材

- 生成或替换人物默认立绘、情绪立绘、受伤立绘前，必须完整阅读并遵守 `docs/presentation-character-asset-spec.md`。
- 默认成品统一为 `1024 × 1536`、竖版 `2:3`、透明 PNG、腰部以上半身像；禁止混用全身、四分之三身、腿部可见或头像特写构图。
- 先验收 `idle` 的景别和构图，再以该图为唯一参考生成状态图；状态图必须保持相同身份、服装、缩放、裁切、头部位置和底部锚点。
- 人物素材路径和绑定只属于 presentation 层，不得写入世界数据库、剧情蓝图或游戏固定上下文。
- 素材登记后，应使用 `pnpm dev` 在内置浏览器的宽屏游戏舞台中与其他人物并排检查视觉比例。

## 剧情任务判定

- 自动任务判定由 `server/questJudge.js` 在玩家回合结束后统一执行；任务的自然语言完成标准和解锁关系放在 `quest` 组件数据中。
- 不要为具体剧情任务增加关键词匹配、场景 ID 判断或任务专属 `if/else`；后端只负责校验模型结构、证据 ID 和持久化。
- 新增或调整自动判定任务时，同步维护 `docs/quest-judge.md` 和 `server/questJudge.test.js`。

## 嵌套 AGENTS.md 规则

- 编辑子目录前，检查该目录或父目录是否存在更具体的 `AGENTS.md` 或 `AGENTS.override.md`；距离目标文件最近的说明优先。
- 只有子目录存在稳定且不同于仓库通用规则的约定时，才新增嵌套 `AGENTS.md`，不要重复根目录规则。

## AGENTS.md 维护

- 当项目命令、仓库结构、AI 工作流或素材生成约定发生稳定变化时，检查并同步更新本文件和对应详细文档。
- 这里只记录可长期复用的仓库规则；不要写入任务日志、临时调试记录、密钥、凭证、私有环境变量或时间戳。
- 提交前确认本文件中的路径、命令和约束仍与项目实际实现一致。
